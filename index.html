Destini.ai — Full Stack Intern
Internship • Oct 2025 — Present
Summary

As a Full Stack Intern at Destini.ai, I contributed to product-facing frontend components and backend integrations to deliver features for both end-users and enterprise clients. Work included building React components, integrating cloud services, implementing secure payment flows, and connecting admin tooling to APIs.

Key responsibilities & achievements
  • Frontend: Implemented and refactored React.js components and application flows to improve UX, performance, and maintainability (state management, routing, lazy loading, form validation).
  • Azure Translation Integration: Integrated Azure Translation services to enable multilingual UI/content. Implemented secure handling of service keys (server-side token exchange & secrets management) and caching of translations for performance.
  • Payout System: Implemented the payouts flow — created frontend screens for payout requests, integrated backend payout APIs, and added client-side validations and error-handling for financial transactions.
  • Enterprise SSO: Integrated enterprise Single Sign-On (SSO) for corporate customers, working with SAML/OAuth endpoints and updating authentication flows to support SSO login and role-based access.
  • Admin Panel: Built and connected the Admin Panel to backend REST APIs for user/product management, audit logs, and configuration controls. Implemented paginated lists, filters, bulk actions, and secure admin endpoints.
  • API & Backend Collaboration: Worked with backend engineers to define API contracts, wrote integration tests for endpoints used by the frontend, and helped debug issues across environments (local → staging → production).
  • Dev Practices: Participated in code reviews, used Git feature branches and pull requests, and helped with CI/CD pipelines for staging deployments.
Components & application flows (examples)
  • Reusable input & form components with validation (Formik-like patterns / custom hooks for handling form state).
  • Dashboard widgets — data fetching, optimistic updates, and client-side caching strategies.
  • Payout request flow: request form → validation → submit → status screen with polling for payout confirmation.
  • SSO flow: SSO button → redirect to identity provider → callback handling → token exchange and session creation.
  • Admin list flows: server-side pagination + client-side filters + bulk status updates with confirmation modals.
Technologies

React.js · JavaScript (ES6+) · Node.js · Express · MongoDB · Azure Translator API · SSO (SAML/OAuth) · REST APIs · Bootstrap · Git · CI/CD

Notes & impact

The features delivered improved platform accessibility (multilingual support), added enterprise-ready auth (SSO), and enabled financial workflows (payout system) required by partners. The Admin Panel improvements reduced manual support effort and sped up content/product management.
